```c
x8 = 0xC122D952  // -0x3edd26ae
while (true) {
    if (x8 > 0xe0ee0cb2) {  // signed compare
        if (x8 != 0xe0ee0cb3 && x8 == 0xe19077be) break;
        if (x8 != 0x7fdcd428) continue;
        // 这条路径：x8 == 0x7fdcd428
        x9_1 = sp;
        var_70 = x9_1 - 0x10;
        sp = x9_1 - 0x10;
        x8 = 0xE00C0E7C;
        x9_1[-2] = arg1;  // *(sp - 0x10) = arg1
        continue;
    }
    if (x8 == 0xad2b5e0d) {
        x8_3 = *var_70;  // = arg1
        // ... 内嵌的多层状态机最终调 free(x8_3) ...
        free(x8_3);
        x8 = 0xE19077BE;
        continue;
    }
    if (x8 != 0xc122d952 && x8 != 0xe00c0e7c) continue;
    x8 = 0xAD2B5E0D;
    continue;  // 其它情况
    x8 = 0x7fdcd428;
    return;
}
```

## 去混淆后伪代码（5 条 HLIL）

```c
sp[-2] = arg1;             // sp 是 int64*，[-2] 即 -0x10 字节偏移
free(*(sp - 0x10));        // 读回刚写进去的 arg1
return;
```

**对外可见副作用序列**：
1. `sp[-2] = arg1` ≡ `*(sp - 0x10) = arg1` ✓
2. `free(*(sp - 0x10))` 读到的就是 step 1 写入的 arg1，所以 ≡ `free(arg1)` ✓
3. `return` ✓

## 结论

**原版与去混淆后的副作用序列逐项相同，逻辑上完全等价**。

实际函数语义：`free(arg1); return;`，原代码用 5 层嵌套状态机把这个简单函数包装得像 49 条 HLIL 那么复杂。
